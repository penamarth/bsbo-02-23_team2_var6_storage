# Рамки: Информационная система склада

## Сценарий: Регистрация отгрузки товаров

## Уровень: Задача, определенная пользователем

## Основной исполнитель: Кладовщик

### Заинтересованные лица и их требования:

- **Кладовщик:**
  - Хочет быстро и точно регистрировать отгрузку товаров со склада
  - Хочет видеть доступные остатки товаров для корректного формирования отгрузки
  - Хочет получать подтверждение об успешной отгрузке с детальной информацией

- **Менеджер склада:**
  - Хочет контролировать процесс отгрузки в реальном времени
  - Хочет минимизировать ошибки при комплектации заказов
  - Хочет иметь возможность отслеживать историю всех отгрузок для анализа эффективности работы склада

- **Система учета:**
  - Хочет получать оперативные данные об отгрузках для актуализации финансовой отчетности
  - Хочет обеспечить корректное списание товаров с баланса

- **Система логистики:**
  - Хочет иметь данные об отгрузках для планирования маршрутов доставки
  - Хочет получать информацию о грузоподъемности и габаритах отгружаемых товаров
  - Хочет интегрироваться с внешними службами доставки

### Предусловия:

- Кладовщик авторизован в системе
- В системе существует заказ на отгрузку товаров
- Товары для отгрузки имеются на складе в достаточном количестве
- Система имеет информацию о местах хранения товаров
- Существует договор с клиентом и реквизиты для оформления документов

### Результаты (Постусловия):

- Товары списаны с остатков склада
- Сформирован документ отгрузки (товарная накладная)
- Обновлена информация о занятости мест хранения
- Данные об отгрузке переданы в систему учета и систему логистики
- Статус заказа изменен на "Отгружен"

### Основной успешный сценарий:

1. Кладовщик в главном меню выбирает раздел "Отгрузка товаров"
   - Система отображает список заказов, ожидающих отгрузки
2. Кладовщик выбирает заказ для отгрузки
   - Система загружает детальную информацию о заказе: состав товаров, количество, клиент, адрес доставки
3. Система показывает рекомендуемые места хранения для сборки заказа
   - Система формирует оптимальный маршрут сборки заказа по складу
4. Кладовщик подтверждает готовность к сборке заказа
   - Система создает задание на сборку и резервирует товары на складе
5. Кладовщик сканирует товары при сборке заказа
   - Система сравнивает фактически отгружаемое количество с заказанным
6. При завершении сборки кладовщик подтверждает отгрузку
   - Система проверяет полноту сборки и соответствие заказу
7. Система формирует документы отгрузки
   - Система обновляет остатки товаров на складе
   - Система освобождает места хранения, с которых были взяты товары
   - Система отправляет данные в систему учета для оформления финансовых документов
   - Система отправляет данные в систему логистики для организации доставки
8. Кладовщик получает подтверждение завершения отгрузки
   - Система отображает сформированную товарную накладную с QR-кодом для идентификации
   - Система отправляет уведомление менеджеру склада и логисту

### Расширения (альтернативные потоки):

- **Недостаточное количество товара на складе:**
> 1. При сборке кладовщик обнаруживает, что фактическое количество товара меньше заказанного  
> 2. Система предлагает варианты:  
>    - Отгрузить доступное количество с пометкой "Частичная отгрузка"  
>    - Отменить отгрузку товара и уведомить менеджера  
>    - Заменить товар на аналогичный (при согласовании с клиентом)  
> 3. Кладовщик выбирает вариант "Частичная отгрузка"  
> 4. Система создает два документа: один для фактической отгрузки, второй для остатка  
> 5. Система изменяет статус основного заказа на "Частично отгружен"  
> 6. Система создает уведомление для менеджера о необходимости согласования с клиентом по остатку заказа

- **Повреждение товара при сборке:**
> 1. Кладовщик обнаруживает повреждение товара при сборке заказа  
> 2. Система предлагает зафиксировать брак и выбрать замену  
> 3. Кладовщик сканирует поврежденный товар и указывает причину брака  
> 4. Система списывает товар как брак и ищет замену на складе  
> 5. При наличии замены система предлагает альтернативное место хранения  
> 6. При отсутствии замены система предлагает частичную отгрузку или замену на аналогичный товар  
> 7. После выбора варианта система обновляет состав заказа и формирует акт о браке

### Специальные требования:

- Интуитивный интерфейс с поддержкой сканеров штрих-кодов для ускорения сборки
- Возможность работы в офлайн-режиме с последующей синхронизацией при восстановлении связи
- Визуальное отображение схемы склада с проложенным маршрутом сборки заказа
- Поддержка единовременной и частичной отгрузки заказов
- Автоматическое формирование всех необходимых документов в соответствии с законодательством
- Интеграция с электронным документооборотом для передачи данных контрагентам
- Валидация данных в реальном времени для предотвращения ошибок
- Аудит всех действий кладовщика с привязкой ко времени и месту выполнения операций

### Список технологий и типов данных:

- **Данные заказа:** номер заказа, клиент, адрес доставки, сроки отгрузки, приоритет, статус
- **Данные товаров:** артикул, наименование, количество, единица измерения, серийные номера, срок годности
- **Места хранения:** зона склада, ряд, стеллаж, полка, ячейка, вместимость
- **Документы:** товарная накладная, акт приемки-передачи, упаковочный лист, спецификация
- **Маршрутизация:** алгоритмы оптимизации маршрутов сборки заказов по складу
- **Интеграция:** API для обмена данными с системой учета (1С, SAP), системой логистики, CRM-системой
- **Уведомления:** push-уведомления, SMS, email для информирования участников процесса

### Частота использования:

- Высокая частота в рабочие дни, особенно в периоды пиковых продаж
- Среднее количество операций отгрузки в день: 50-100 для среднего склада
- Периоды пиковой нагрузки: конец месяца, предпраздничные дни, сезонные распродажи
